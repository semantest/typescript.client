Semantest Documentation

Installation

Semantest can be installed via npm or used directly from the browser extension.

NPM Installation

# Install core packages
npm install @semantest/browser @semantest/nodejs.server

# Install platform-specific packages
npm install @semantest/chatgpt.com
npm install @semantest/claude.ai
npm install @semantest/google.com

# Install TypeScript-EDA foundation
npm install typescript-eda-domain typescript-eda-infrastructure

Chrome Extension

Install the Semantest Chrome Extension from the Chrome Web Store for browser automation.

Quick Start

Get up and running with Semantest in minutes.

Basic Usage

import { SemanTestClient } from '@semantest/browser';
import { ChatGPTClient } from '@semantest/chatgpt.com';

async function main() {
    // Initialize the client
    const client = new ChatGPTClient({
        polling: {
            interval: 1000,
            maxRetries: 3
        }
    });

    // Connect to ChatGPT
    await client.initialize();

    // Send a prompt
    const response = await client.sendPrompt(
        'Generate a Python function to calculate fibonacci'
    );

    // Get the response
    console.log('ChatGPT Response:', response.content);

    // Clean up
    await client.cleanup();
}

main().catch(console.error);

Core Concepts

Semantic Contracts

Semantic contracts define what an application can do, not how it does it. This abstraction makes your automation resilient to UI changes.

Event-Driven Architecture

All operations in Semantest flow through domain events, ensuring loose coupling and high scalability.

Domain-Based Routing

Semantest automatically routes requests to the appropriate AI platform based on the domain in the event payload.

API Reference

SemanTestClient

class SemanTestClient {
    constructor(config: ClientConfig)
    async initialize(): Promise
    async sendPrompt(prompt: string, options?: PromptOptions): Promise
    async cleanup(): Promise
    on(event: string, handler: Function): void
    off(event: string, handler: Function): void
}

Events

  • connected - Fired when connection is established
  • promptSent - Fired when a prompt is sent
  • responseReceived - Fired when a response is received
  • error - Fired when an error occurs
  • disconnected - Fired when connection is closed

Examples

Multi-Platform Integration

// Work with multiple AI platforms simultaneously
import { ChatGPTClient } from '@semantest/chatgpt.com';
import { ClaudeClient } from '@semantest/claude.ai';
import { GeminiClient } from '@semantest/google.com';

async function compareAIResponses(prompt: string) {
    const [chatgptResponse, claudeResponse, geminiResponse] = await Promise.all([
        new ChatGPTClient().sendPrompt(prompt),
        new ClaudeClient().sendPrompt(prompt),
        new GeminiClient().sendPrompt(prompt)
    ]);

    return {
        chatgpt: chatgptResponse.content,
        claude: claudeResponse.content,
        gemini: geminiResponse.content
    };
}

Event Handling

// Listen to events for monitoring and debugging
client.on('promptSent', (event) => {
    console.log('Prompt sent:', event.prompt);
    metrics.record('prompt.sent', event);
});

client.on('responseReceived', (event) => {
    console.log('Response received:', event.content);
    metrics.record('response.received', event);
});

client.on('error', (error) => {
    console.error('Error occurred:', error);
    alerting.notify('semantest.error', error);
});

Troubleshooting

Common Issues

  • Connection Failed

    Ensure the Chrome extension is installed and the polling server is running.

  • No Response Received

    Check that the target AI platform tab is open and logged in.

  • Domain Not Recognized

    Verify the domain field is included in your event payload.
